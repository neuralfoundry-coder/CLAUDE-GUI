# 5. 유스케이스

## UC-01: 프로젝트 열기 및 탐색

| 항목 | 내용 |
|------|------|
| **액터** | 개발자 |
| **사전조건** | ClaudeGUI 서버 실행 중, 프로젝트 디렉토리 설정 완료 |
| **관련 FR** | FR-200, FR-300, FR-900 |

### 기본 흐름

1. 사용자가 브라우저에서 ClaudeGUI에 접속한다.
2. 좌측 파일 탐색기에 프로젝트 디렉토리 트리가 표시된다.
3. 사용자가 폴더를 클릭하여 하위 항목을 펼친다.
4. 파일을 클릭하면 중앙 에디터에 새 탭으로 열린다.
5. 에디터에 파일 내용이 구문 강조와 함께 표시된다.

### 대안 흐름

- **3a.** `Cmd+P`로 파일명 검색하여 빠르게 열기
- **4a.** 이미 열린 파일의 경우 기존 탭으로 포커스 전환
- **5a.** 프리뷰 가능한 파일(HTML, MD, PDF)인 경우 우측 프리뷰 패널에도 자동 표시

### 사후조건

- 선택한 파일이 에디터 탭에 열려있다.
- 파일 탐색기에서 해당 파일이 하이라이트되어 있다.

## UC-02: Claude에게 코드 수정 요청

| 항목 | 내용 |
|------|------|
| **액터** | 개발자 |
| **사전조건** | Claude CLI 인증 완료, Claude 세션 활성 |
| **관련 FR** | FR-500, FR-300, FR-907 |

### 기본 흐름

1. 사용자가 Claude 입력란에 자연어 명령을 입력한다.
   - 예: "login 함수에 에러 핸들링을 추가해줘"
2. 시스템이 Agent SDK를 통해 Claude에 쿼리를 전송한다.
3. Claude의 스트리밍 응답이 실시간으로 표시된다 (타이핑 효과).
4. Claude가 파일 수정 도구를 호출한다.
5. 권한 요청 모달이 표시된다 (FR-505 참조, UC-03).
6. 사용자가 승인하면 Claude가 파일을 수정한다.
7. chokidar가 파일 변경을 감지하여 에디터에 반영한다.
8. 에디터에 diff 뷰로 변경사항이 표시된다.
9. 사용자가 변경사항을 수락 또는 거절한다.
10. 쿼리 완료 후 비용/토큰 사용량이 표시된다.

### 대안 흐름

- **5a.** 자동 승인 규칙에 해당하면 모달 없이 자동 진행
- **7a.** 에디터에 미저장 변경이 있으면 충돌 알림 표시
- **9a.** 부분 수락 — 특정 hunk만 수락하고 나머지 거절

### 사후조건

- 수락된 변경사항이 파일에 적용되어 있다.
- Claude 세션에 대화 기록이 남아있다.

## UC-03: 권한 승인/거부

| 항목 | 내용 |
|------|------|
| **액터** | 개발자 |
| **사전조건** | Claude가 도구 실행을 요청한 상태 |
| **관련 FR** | FR-505, FR-506 |

### 기본 흐름

1. Claude가 파일 쓰기 또는 Bash 명령 실행을 요청한다.
2. 시스템이 권한 요청을 인터셉트하여 GUI 모달을 표시한다.
3. 모달에 도구 이름, 대상 경로/명령어가 표시된다.
4. 사용자가 **승인(Approve)** 버튼을 클릭한다.
5. 시스템이 Agent SDK에 승인 응답을 전달한다.
6. Claude가 요청된 도구를 실행한다.

### 대안 흐름

- **4a.** 사용자가 **거부(Deny)** 클릭 → Claude에 거부 응답 전달 → Claude가 대안 방안 제시
- **2a.** `.claude/settings.json` 화이트리스트에 해당 → 자동 승인, "Auto-approved" 배지 표시
- **3a.** 위험 명령(rm -rf, 시스템 파일 수정 등)은 경고 아이콘과 함께 강조 표시

### 사후조건

- 승인/거부 결과가 Claude 세션에 기록된다.
- 승인된 경우 도구가 실행 완료된다.

## UC-04: HTML 프레젠테이션 생성 및 편집

| 항목 | 내용 |
|------|------|
| **액터** | 개발자 |
| **사전조건** | Claude 세션 활성 |
| **관련 FR** | FR-700, FR-600 |

### 기본 흐름

1. 사용자가 Claude에게 프레젠테이션 생성을 요청한다.
   - 예: "프로젝트 아키텍처 소개 프레젠테이션 5장짜리 만들어줘"
2. Claude가 reveal.js 형식의 HTML 파일을 생성한다.
3. 우측 프리뷰 패널에 슬라이드가 렌더링된다.
4. 사용자가 자연어로 수정을 요청한다.
   - 예: "3번 슬라이드에 시퀀스 다이어그램 추가해줘"
5. Claude가 해당 슬라이드의 HTML을 수정한다.
6. iframe 내 DOM이 패치되고 `Reveal.sync()`가 호출된다.
7. 수정된 슬라이드가 리로드 없이 즉시 반영된다.
8. 사용자가 PPTX 내보내기를 요청한다.
9. PptxGenJS가 `.pptx` 파일을 생성하여 다운로드한다.

### 대안 흐름

- **3a.** 에디터에서 직접 HTML/Markdown을 편집 → 프리뷰 실시간 갱신
- **8a.** PDF 내보내기 선택 → DeckTape 또는 print-pdf 방식으로 생성
- **4a.** 프리뷰에서 특정 슬라이드 클릭 → 에디터가 해당 코드로 스크롤 (양방향 동기화)

### 사후조건

- 프레젠테이션 파일이 프로젝트에 저장되어 있다.
- 내보낸 파일(PPTX/PDF)이 다운로드되었다.

## UC-05: 멀티포맷 문서 프리뷰

| 항목 | 내용 |
|------|------|
| **액터** | 개발자 |
| **사전조건** | 프리뷰 가능한 파일이 프로젝트에 존재 |
| **관련 FR** | FR-600 |

### 기본 흐름

1. 사용자가 파일 탐색기에서 파일을 선택한다.
2. 시스템이 파일 확장자를 기반으로 타입을 감지한다.
3. 적절한 렌더러가 프리뷰 패널에 로드된다.
   - `.html` → iframe srcdoc 렌더링
   - `.pdf` → react-pdf 페이지 뷰
   - `.md` → react-markdown 렌더링
   - `.png/.jpg` → 이미지 뷰어 (줌/팬)
4. 에디터에서 파일을 수정하면 프리뷰가 300ms 디바운스 후 갱신된다.

### 대안 흐름

- **2a.** 지원하지 않는 파일 타입 → "프리뷰 불가" 메시지 표시
- **3a.** PDF: 페이지 네비게이션 (이전/다음, 페이지 점프)
- **4a.** HTML: CSS만 변경된 경우 `postMessage`로 스타일만 업데이트

### 사후조건

- 선택한 파일의 프리뷰가 우측 패널에 표시되어 있다.

## UC-06: 터미널 직접 사용

| 항목 | 내용 |
|------|------|
| **액터** | 개발자 |
| **사전조건** | 터미널 패널 활성 |
| **관련 FR** | FR-400 |

### 기본 흐름

1. 사용자가 `Cmd+J`로 터미널 패널을 연다 (또는 클릭).
2. 터미널이 프로젝트 디렉토리에서 셸 세션을 시작한다.
3. 사용자가 명령어를 입력한다 (예: `npm test`).
4. 명령어 출력이 ANSI 컬러와 함께 실시간 렌더링된다.
5. 명령어가 파일을 수정하면 chokidar가 감지하여 에디터/파일트리를 갱신한다.

### 대안 흐름

- **2a.** 추가 터미널 세션 생성 (탭 또는 분할)
- **3a.** `claude` 명령어 직접 입력 → CLI가 터미널에서 직접 실행
- **4a.** 대량 출력 시 배압 제어 활성화 (50MB 한도)
- **5a.** `Ctrl+F`로 터미널 버퍼 내 텍스트 검색

### 사후조건

- 명령어 실행 결과가 터미널에 표시되어 있다.
- 파일 변경이 있었다면 에디터와 파일 트리에 반영되어 있다.

## UC-07: 세션 관리

| 항목 | 내용 |
|------|------|
| **액터** | 개발자 |
| **사전조건** | ClaudeGUI 서버 실행 중 |
| **관련 FR** | FR-503, FR-504 |

### 기본 흐름

1. 사용자가 세션 관리 UI를 연다.
2. 기존 세션 목록이 표시된다 (이름, 생성일, 마지막 사용, 비용).

- **3b.** 세션 이름 변경
- **3c.** 세션 삭제 (확인 후)

### 사후조건

- 선택한 세션이 활성화되어 Claude와 대화 가능한 상태이다.

## UC-08: 레이아웃 커스터마이징

| 항목 | 내용 |
|------|------|
| **액터** | 개발자 |
| **사전조건** | ClaudeGUI 접속 상태 |
| **관련 FR** | FR-100 |

### 기본 흐름

1. 사용자가 패널 경계의 드래그 핸들을 조작하여 패널 크기를 변경한다.
2. 변경된 크기가 즉시 적용된다.
3. 레이아웃 상태가 `localStorage`에 자동 저장된다.

### 대안 흐름

- **1a.** `Cmd+B`로 사이드바(파일 탐색기) 토글
- **1b.** `Cmd+J`로 터미널 패널 토글
- **1c.** 패널 헤더의 접기 버튼으로 패널 접기/펼치기
- **3a.** 브라우저 새로고침 시 저장된 레이아웃이 복원된다

### 사후조건

- 사용자가 조정한 레이아웃이 유지되며, 다음 접속 시에도 동일하게 표시된다.
